诊断：查看当前驱动加载状态

在配置自动加载之前，先摸清楚当前系统哪些驱动已经加载、哪些还缺失。在开发板上执行以下命令：

① 查看已加载的内核模块

bash开发板上执行
lsmod

② 检查各外设是否已识别

bash开发板上执行
ip link show                          # 查看网卡列表（有无 wlan0、eth0、eth1）
ls /sys/class/net/                    # 同上
ls /sys/class/bluetooth/              # 是否有蓝牙设备
ls /dev/dri/                          # DRM GPU 设备（应有 card0、renderD128）
ls /dev/video*                        # 摄像头设备
ls /proc/asound/cards                 # 音频声卡列表
lspci                                 # PCIe 设备列表
lsusb                                 # USB 设备列表

③ 查看内核识别到哪些模块

先确认内核模块目录中有哪些模块可用：

bash开发板上执行
ls /lib/modules/$(uname -r)/kernel/drivers/

搜索特定模块是否存在：

bash开发板上执行
find /lib/modules/$(uname -r)/ -name "*.ko" | grep -E "mali|gpu|wifi|rtl|8733|gmac|motorcomm|yt8531|es8388|mpp|vcodec|rkisp" | sort

④ 查看开机时的内核日志

检查驱动加载和设备探测日志，看哪些设备有报错：

bash开发板上执行
dmesg | grep -E "mali|gpu|wifi|rtl8|gmac|motorcomm|yt8531|es8388|mpp|vcodec|rkisp|bluetooth|hci" | head -60
💡
分析技巧：如果 dmesg 中出现 module not found 或 No such file，说明模块文件缺失，需要重新从 SDK 内核编译安装。如果出现 firmware not found，说明固件文件缺失，需要复制固件。

配置 modules-load.d 自动加载

Linux 系统通过 /etc/modules-load.d/ 目录下的配置文件，让 systemd-modules-load 服务在开机时自动加载指定模块。这是 Ubuntu 22.04 推荐的标准方式。

创建 RK3588 驱动自动加载配置文件

在开发板上创建以下配置文件，一次性配置所有板载外设的驱动自动加载：

写入文件/etc/modules-load.d/rk3588-drivers.conf
sudo tee /etc/modules-load.d/rk3588-drivers.conf << 'EOF'
# ============================================
# ATK-DLRK3588 板载外设驱动自动加载配置
# ============================================

# --- GPU / 显示 ---
bifrost_kbase
rockchipdrm
dw_hdmi_cec

# --- 以太网 ---
rk_gmac_dwmac
motorcomm

# --- WiFi (RTL8733BU USB) ---
rtl8xxxu

# --- 蓝牙 ---
btusb
bluetooth
hci_uart

# --- 音频 ---
snd_soc_es8388
snd_soc_rockchip_i2s

# --- 视频编解码 (MPP/VPU) ---
rk_vcodec

# --- NPU (RKNPU2) ---
rknpu

# --- 摄像头 ISP ---
video_rkisp
rkisp_hw

# --- 其他外设 ---
pwm_rockchip
rockchip_saradc
rtc_hym8563
EOF

使配置立即生效（不重启验证）：

bash开发板上执行
sudo systemctl restart systemd-modules-load.service
sudo systemctl status systemd-modules-load.service

配置 modprobe 模块参数（可选）

部分模块加载时需要传入参数，在 /etc/modprobe.d/ 下创建配置：

写入文件/etc/modprobe.d/rk3588.conf
sudo tee /etc/modprobe.d/rk3588.conf << 'EOF'
# RK3588 模块参数配置
# Mali GPU - 设置性能模式
options bifrost_kbase power_policy=coarse_demand

# 蓝牙 - 启用 HCI UART 协议
options hci_uart enable_autobaud=1
EOF
⚠️
注意：配置文件中的模块名必须与实际 .ko 文件名完全一致（不含 .ko 后缀）。如果某个模块不存在，systemd-modules-load 会在日志中报错但不影响其他模块加载。

WiFi / 蓝牙驱动配置

ATK-DLRK3588 板载 WiFi/BT 模块为 RTL8733BU，通过 USB 内部总线连接。Ubuntu 内核通常已内置 rtl8xxxu 驱动，但需要确认固件文件存在。

① 检查 WiFi 模块识别情况

bash开发板上执行
lsusb | grep -i realtek
lsmod | grep rtl

② 安装 WiFi 固件

Ubuntu 22.04 的 linux-firmware 包通常包含 RTL 系列固件，先确认是否已安装：

bash开发板上执行
dpkg -l linux-firmware
ls /lib/firmware/rtlwifi/ 2>/dev/null || echo "固件目录不存在"

如果固件缺失，安装 linux-firmware 包：

bash开发板上执行
sudo apt update
sudo apt install -y linux-firmware wireless-tools wpasupplicant

③ 手动测试 WiFi 驱动加载

bash开发板上执行
sudo modprobe rtl8xxxu
ip link show wlan0

如果 rtl8xxxu 无法驱动该模块，尝试 SDK 中提供的树外驱动：

bash开发板上执行
find /lib/modules/$(uname -r)/ -name "*8733*" -o -name "*rtl8733*" 2>/dev/null
sudo modprobe 8733bu 2>/dev/null || sudo modprobe rtl8733bu 2>/dev/null

④ 配置 WiFi 开机自动连接

bash开发板上执行
sudo nmcli dev wifi connect "你的WiFi名称" password "你的WiFi密码"
sudo nmcli con modify "你的WiFi名称" connection.autoconnect yes

⑤ 蓝牙驱动配置

安装蓝牙协议栈并启用服务：

bash开发板上执行
sudo apt install -y bluez bluez-tools rfkill

解除蓝牙 rfkill 软封锁并启用服务：

bash开发板上执行
sudo rfkill unblock bluetooth
sudo rfkill unblock all
sudo systemctl enable bluetooth
sudo systemctl start bluetooth

验证蓝牙是否正常工作：

bash开发板上执行
hciconfig -a
bluetoothctl show

⑥ 创建 rfkill 解锁开机服务

防止 rfkill 在开机时阻止蓝牙和 WiFi：

写入文件/etc/systemd/system/rfkill-unblock.service
sudo tee /etc/systemd/system/rfkill-unblock.service << 'EOF'
[Unit]
Description=Unblock all RF devices on boot
After=network-pre.target
Before=network.target

[Service]
Type=oneshot
ExecStart=/usr/sbin/rfkill unblock all
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
EOF
bash开发板上执行
sudo systemctl daemon-reload
sudo systemctl enable rfkill-unblock.service
sudo systemctl start rfkill-unblock.service

GPU / 显示驱动配置

RK3588 搭载 Mali-G610 MP4 GPU，需要正确加载内核驱动模块以及用户态 OpenGL/Vulkan 库。

① 确认 GPU 模块名称

bash开发板上执行
find /lib/modules/$(uname -r)/ -name "*.ko" | grep -E "mali|gpu|bifrost|panfrost"

根据内核版本，模块名可能是：

内核版本模块名说明
RK Linux 5.10.x (SDK)bifrost_kbaseArm 官方 Mali 驱动
上游内核 6.xpanfrost开源 GPU 驱动（功能受限）
部分 SDK 变体mali_kbase旧版命名

② 加载 GPU 模块并验证

bash开发板上执行
sudo modprobe bifrost_kbase
ls /dev/mali*
ls /dev/dri/

成功后应看到 /dev/mali0 以及 /dev/dri/card0、/dev/dri/renderD128。

③ 安装用户态 GPU 库

安装 Mesa 开源图形库（HDMI/DP 显示基础）：

bash开发板上执行
sudo apt install -y libdrm2 libgl1-mesa-dri libgles2-mesa-dev libegl1-mesa-dev

如果使用正点原子 SDK 提供的 Mali 闭源用户态库，从宿主机 buildroot target 复制过来：

bash宿主机执行（将文件 scp 到板子）
scp ~/rk3588_sdk/buildroot/output/rockchip_atk_dlrk3588/target/usr/lib/aarch64-linux-gnu/libmali*.so* \
    ubuntu@开发板IP:/tmp/
bash开发板上执行
sudo cp /tmp/libmali*.so* /usr/lib/aarch64-linux-gnu/
sudo ldconfig

④ 配置 DRM 权限

bash开发板上执行
sudo usermod -aG video,render $(whoami)
sudo chmod 666 /dev/dri/renderD128
sudo chmod 666 /dev/mali0 2>/dev/null || true

为了永久生效，创建 udev 规则：

写入文件/etc/udev/rules.d/99-mali-permissions.rules
sudo tee /etc/udev/rules.d/99-mali-permissions.rules << 'EOF'
SUBSYSTEM=="drm", GROUP="video", MODE="0666"
KERNEL=="mali*", GROUP="video", MODE="0666"
KERNEL=="renderD*", GROUP="render", MODE="0666"
EOF
bash开发板上执行
sudo udevadm control --reload-rules
sudo udevadm trigger

以太网驱动配置

ATK-DLRK3588 的千兆以太网由两部分组成：RK3588 内置的 GMAC MAC 控制器（驱动 rk_gmac_dwmac，通常由设备树自动探测）和外置 PHY 芯片 MotorComm YT8531C（驱动模块 motorcomm，需要手动确认加载）。

① 检查以太网状态

查看当前网卡列表和已加载的相关模块：

bash开发板上执行
ip link show
lsmod | grep -E "gmac|motorcomm|yt8531"
dmesg | grep -E "eth|gmac|motorcomm|yt8531|YT8531"

② 确认 motorcomm PHY 驱动已安装

在 SDK 编译的内核中，YT8531C PHY 驱动以 motorcomm.ko 形式存在（内核配置项 CONFIG_MOTORCOMM_PHY），先确认模块文件存在：

bash开发板上执行
find /lib/modules/$(uname -r)/ -name "motorcomm*"

如果找到了模块文件，手动加载验证：

bash开发板上执行
sudo modprobe motorcomm
lsmod | grep motorcomm
dmesg | tail -20

加载成功后，GMAC 会自动探测到 YT8531C PHY，网卡 eth0 应出现在 ip link show 结果中。

💡
驱动内置情况：如果 find 命令找不到 motorcomm.ko，说明该 PHY 驱动被静态编译进内核（CONFIG_MOTORCOMM_PHY=y），此时无需手动 modprobe，设备树会自动完成绑定，ip link show 里直接能看到 eth0。

③ 配置开机自动获取 IP（netplan）

使用 Ubuntu 22.04 标准的 netplan 方式配置网卡 DHCP：

写入文件/etc/netplan/01-rk3588-network.yaml
sudo tee /etc/netplan/01-rk3588-network.yaml << 'EOF'
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: true
      optional: true
      nameservers:
        addresses: [114.114.114.114, 8.8.8.8]
EOF
bash开发板上执行
sudo chmod 600 /etc/netplan/01-rk3588-network.yaml
sudo netplan apply

验证网络是否通：

bash开发板上执行
ip addr show eth0
ping -c 3 114.114.114.114

① 检查音频设备

bash开发板上执行
cat /proc/asound/cards

NPU（RKNPU2）驱动配置

RK3588 内置 6 TOPS 算力的 NPU，驱动由两层组成：内核层 rknpu.ko 负责硬件调度，用户态层 librknnrt.so 提供推理 API。两层都需要正确安装，NPU 才能使用。

① 检查内核驱动是否已加载

先确认内核模块文件存在，并查看当前加载状态：

bash开发板上执行
find /lib/modules/$(uname -r)/ -name "rknpu*"
lsmod | grep rknpu
ls /dev/rknpu*

如果模块未加载，手动 modprobe 验证：

bash开发板上执行
sudo modprobe rknpu
dmesg | grep -i rknpu | tail -10

加载成功后，dmesg 中应出现类似如下输出：

内核日志（示例）预期输出
rknpu: Initialized rknpu 0.9.x 20240xxx for ffe40000.npu on minor 0
rknpu: NPU power on.
💡
驱动内置说明：正点原子 SDK 内核通常将 CONFIG_ROCKCHIP_RKNPU=y 静态编译进内核，此时 find 找不到 rknpu.ko 属于正常现象，/dev/rknpu0 会在启动时自动出现，无需手动 modprobe。

② 配置 /dev/rknpu 设备权限

默认情况下 /dev/rknpu0 只有 root 可访问，需要添加 udev 规则让普通用户也能使用 NPU：

写入文件/etc/udev/rules.d/99-rknpu.rules
sudo tee /etc/udev/rules.d/99-rknpu.rules << 'EOF'
KERNEL=="rknpu*", MODE="0666"
KERNEL=="mpp_service", MODE="0666"
EOF
bash开发板上执行 — 立即生效
sudo udevadm control --reload-rules
sudo udevadm trigger
ls -la /dev/rknpu*

③ 安装用户态运行库 librknnrt.so

从正点原子 SDK 的 buildroot 输出目录中复制预编译好的运行库（无需重新编译）：

bash宿主机执行 — 复制库文件到 Ubuntu rootfs
BUILDROOT_TARGET=~/rk3588_sdk/buildroot/output/rockchip_atk_dlrk3588/target

# 复制 RKNN Runtime 库
sudo cp ${BUILDROOT_TARGET}/usr/lib/librknnrt.so \
    ~/ubuntu_build/ubuntu_rootfs/usr/lib/aarch64-linux-gnu/

# 复制 rknn_server 可执行文件
sudo cp ${BUILDROOT_TARGET}/usr/bin/rknn_server \
    ~/ubuntu_build/ubuntu_rootfs/usr/bin/

sudo chmod +x ~/ubuntu_build/ubuntu_rootfs/usr/bin/rknn_server

如果 Buildroot 输出中没有现成的库，也可以从 RKNN-Toolkit2 仓库获取预编译版本：

bash开发板上执行（需要网络）
# 克隆 RKNN-Toolkit2，找到 RK3588 对应的运行库
git clone --depth=1 https://github.com/airockchip/rknn-toolkit2.git /tmp/rknn-toolkit2

# 复制 librknnrt.so 到系统库目录
sudo cp /tmp/rknn-toolkit2/rknpu2/runtime/Linux/librknn_api/aarch64/librknnrt.so \
    /usr/lib/aarch64-linux-gnu/
sudo ldconfig

验证库已正确安装：

bash开发板上执行
ldconfig -p | grep librknnrt
strings /usr/lib/aarch64-linux-gnu/librknnrt.so | grep "librknnrt version"

④ 配置 rknn_server 开机自启服务

rknn_server 是 NPU 的后台管理进程，建议配置为 systemd 服务开机自动启动：

写入文件/etc/systemd/system/rknn-server.service
sudo tee /etc/systemd/system/rknn-server.service << 'EOF'
[Unit]
Description=Rockchip NPU RKNN Server
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/rknn_server
Restart=on-failure
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF
bash开发板上执行 — 启用并启动服务
sudo systemctl daemon-reload
sudo systemctl enable rknn-server.service
sudo systemctl start rknn-server.service
sudo systemctl status rknn-server.service

⑤ 验证 NPU 可用

重启后运行以下命令，完整验证 NPU 是否正常工作：

bash开发板上执行
# 确认设备节点存在
ls -la /dev/rknpu*

# 确认内核驱动加载（静态编译时 lsmod 不显示，但设备节点存在即正常）
dmesg | grep -i rknpu

# 确认运行库版本
strings /usr/lib/aarch64-linux-gnu/librknnrt.so | grep "librknnrt version"

# 确认 rknn_server 运行中
systemctl is-active rknn-server.service

创建一键驱动加载启动服务

将所有驱动加载、权限设置、rfkill 解锁等操作整合到一个 systemd 服务中，确保每次开机全自动完成。

① 创建驱动加载脚本

/usr/local/bin/rk3588-load-drivers.sh 
#!/bin/bash
# ATK-DLRK3588 板载驱动自动加载脚本
# 每次开机由 systemd 自动执行

LOG="/var/log/rk3588-drivers.log"
echo "=== $(date) === RK3588 驱动加载开始 ===" >> $LOG

load_module() {
    local mod=$1
    if modinfo "$mod" &>/dev/null; then
        modprobe "$mod" && echo "[OK]  $mod" >> $LOG || echo "[ERR] $mod" >> $LOG
    else
        echo "[SKP] $mod (模块不存在)" >> $LOG
    fi
}

# ====== GPU ======
load_module bifrost_kbase
load_module mali_kbase

# ====== 显示 ======
load_module rockchipdrm
load_module dw_hdmi_cec

# ====== 以太网 ======
load_module rk_gmac_dwmac
load_module motorcomm

# ====== WiFi ======
load_module rtl8xxxu
load_module 8733bu

# ====== 蓝牙 ======
load_module btusb
load_module bluetooth
load_module hci_uart

# ====== 音频 ======

[Unit]

DefaultDependencies=no

Before=network-pre.target bluetooth.target sound.target

[Service]

StandardOutput=journal

② 常见问题排查

现象原因解决方案
WiFi 无 wlan0 接口 RTL8733BU 驱动未加载 / 固件缺失 执行 lsusb 确认USB识别；安装 linux-firmware；检查 dmesg | grep rtl
蓝牙设备不存在 rfkill 封锁 / btusb 未加载 rfkill unblock all；modprobe btusb；检查 dmesg | grep -i bluetooth
/dev/mali0 不存在 Mali 内核模块未加载 find /lib/modules/$(uname -r)/ -name "*mali*"；确认模块名后加入 modules-load.d
HDMI 无输出 rockchipdrm / dw_hdmi 未加载 dmesg | grep -i hdmi；modprobe rockchipdrm；确认 DRM 设备 ls /dev/dri/
声卡不存在 snd_soc_es8388 未加载 modprobe snd_soc_es8388；检查 I2C 设备 ls /dev/i2c-*
模块加载报错 "not found" 模块文件不在当前内核目录中 重新执行 make modules_install 安装内核模块；确认 uname -r 与编译版本一致
eth0 无法获取 IP NetworkManager 未管理有线网卡 修改 /usr/lib/NetworkManager/conf.d/10-globally-managed-devices.conf，添加 except:type:ethernet
