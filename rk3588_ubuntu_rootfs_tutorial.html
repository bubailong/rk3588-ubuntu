编译 Buildroot（获取内核/U-Boot/设备树）

即使我们要替换 rootfs，也需要先通过 SDK 正常编译一次，以获得可用的 内核镜像（boot.img）、 U-Boot（MiniLoaderAll.bin / uboot.img） 以及 交叉编译工具链。

全量编译（推荐首次执行）

bash
cd ~/rk3588_sdk

# 初始化 Buildroot 配置（正点原子 ATK-DLRK3588 专用 defconfig）
cd buildroot
make rockchip_atk_dlrk3588_defconfig

# 开始编译（-j$(nproc) 多核并行，首次耗时 1-3 小时）
make -j$(nproc) 2>&1 | tee build.log
⚠️
编译提示：首次编译会自动下载工具链和源码（约 5-10 GB），请确保网络畅通并耐心等待。 如果下载失败，可手动将依赖包放入 buildroot/dl/ 目录后重新编译。

或者：只编译内核和 U-Boot

bash
cd ~/rk3588_sdk

# 编译 U-Boot
cd u-boot
make CROSS_COMPILE=aarch64-linux-gnu- rk3588_defconfig
make CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)

# 返回 SDK 根目录编译内核
cd ../kernel
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- \
     rockchip_linux_defconfig
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- \
     -j$(nproc) Image modules dtbs

创建挂载/卸载脚本

~/ubuntu_build/mount-rootfs.sh 
#!/bin/bash
# Ubuntu rootfs 挂载/卸载脚本
# 用法: sudo ./mount-rootfs.sh -m <rootfs_dir>  (挂载)
#       sudo ./mount-rootfs.sh -u <rootfs_dir>  (卸载)

mnt() {
    ROOTFS=$1
    echo "[+] Mounting virtual filesystems to ${ROOTFS}..."
    sudo mount -t proc  /proc      ${ROOTFS}/proc
    sudo mount -t sysfs /sys       ${ROOTFS}/sys
    sudo mount -o bind  /dev       ${ROOTFS}/dev
    sudo mount -o bind  /dev/pts   ${ROOTFS}/dev/pts
    echo "[+] Done."
}

umnt() {
    ROOTFS=$1
    echo "[-] Unmounting virtual filesystems from ${ROOTFS}..."
    sudo umount -l ${ROOTFS}/proc    2>/dev/null
    sudo umount -l ${ROOTFS}/sys     2>/dev/null
    sudo umount -l ${ROOTFS}/dev/pts 2>/dev/null
    sudo umount -l ${ROOTFS}/dev     2>/dev/null
    echo "[-] Done."
}

if   [ "$1" = "-m" ] && [ -n "$2" ]; then mnt "$2"
elif [ "$1" = "-u" ] && [ -n "$2" ]; then umnt "$2"
else
    echo "Usage: $0 [-m|-u] <rootfs_dir>"
    exit 1
fi
bash
chmod +x mount-rootfs.sh

# 挂载虚拟文件系统
sudo ./mount-rootfs.sh -m ubuntu_rootfs/

# 进入 chroot 环境（ARM64 on x86_64）
sudo chroot ubuntu_rootfs/ /bin/bash

安装 Rockchip 硬件驱动与内核模块

Ubuntu-Base 本身不含 RK3588 的硬件驱动，需要从 SDK 内核编译结果中复制内核模块到 rootfs。

① 安装内核模块到 Ubuntu rootfs

首先进入 SDK 目录，读取当前内核版本号：

bash宿主机执行

KVER=$(cat kernel/include/config/kernel.release)
echo "内核版本: ${KVER}"

     CROSS_COMPILE=aarch64-linux-gnu- \
     INSTALL_MOD_PATH=~/ubuntu_build/ubuntu_rootfs \
     modules_install

确认模块已成功安装到 rootfs：

bash宿主机执行
ls ~/ubuntu_build/ubuntu_rootfs/lib/modules/

② 复制硬件固件（firmware）

先定义两个路径变量，方便后续操作：

BUILDROOT_TARGET=~/rk3588_sdk/buildroot/output/rockchip_atk_dlrk3588/target
UBUNTU_ROOTFS=~/ubuntu_build/ubuntu_rootfs

复制无线网卡、蓝牙等硬件固件文件：

bash宿主机执行
sudo cp -r ${BUILDROOT_TARGET}/lib/firmware ${UBUNTU_ROOTFS}/lib/

bash宿主机执行 · 图形界面必需
sudo cp -r ${BUILDROOT_TARGET}/usr/lib/aarch64-linux-gnu/libmali* \
    ${UBUNTU_ROOTFS}/usr/lib/aarch64-linux-gnu/ 2>/dev/null || true

③ 配置 fstab（文件系统挂载表）

将以下内容写入 Ubuntu rootfs 的 /etc/fstab，定义系统启动时的分区挂载规则：

sudo tee ~/ubuntu_build/ubuntu_rootfs/etc/fstab << 'EOF'

打包 Ubuntu rootfs 为 ext4 镜像

确认卸载所有挂载点

bash (宿主机)
cd ~/ubuntu_build
sudo ./mount-rootfs.sh -u ubuntu_rootfs/

# 确认无残留挂载
mount | grep ubuntu_rootfs

使用打包脚本制作 rootfs.img

~/ubuntu_build/mk-ubuntu-rootfs.sh 
#!/bin/bash
# 将 ubuntu_rootfs 目录打包为 rootfs.img (ext4)
set -e

ROOTFS_DIR="$(pwd)/ubuntu_rootfs"
OUTPUT_IMG="$(pwd)/ubuntu-rootfs.img"
# 镜像大小（单位 MB）：命令行系统约 2GB，含桌面约 8GB
IMG_SIZE_MB=8192

echo "===================================="
echo " RK3588 Ubuntu rootfs 打包脚本"
echo "===================================="
echo " 源目录: ${ROOTFS_DIR}"
echo " 输出:   ${OUTPUT_IMG}"
echo " 大小:   ${IMG_SIZE_MB} MB"
echo "===================================="

# 检查 qemu-aarch64-static 是否已删除
if [ -f "${ROOTFS_DIR}/usr/bin/qemu-aarch64-static" ]; then
    echo "[!] 发现 qemu-aarch64-static，正在删除..."
    sudo rm -f "${ROOTFS_DIR}/usr/bin/qemu-aarch64-static"
fi

# 步骤1：创建空镜像文件
echo "[1/4] 创建 ${IMG_SIZE_MB}MB 空镜像..."
dd if=/dev/zero of="${OUTPUT_IMG}" bs=1M count=${IMG_SIZE_MB} status=progress

# 步骤2：格式化为 ext4
echo "[2/4] 格式化为 ext4..."
mkfs.ext4 -L "ubuntu-rootfs" "${OUTPUT_IMG}"

# 步骤3：挂载并复制文件
echo "[3/4] 复制根文件系统..."
MOUNT_POINT=$(mktemp -d)
sudo mount -t ext4 "${OUTPUT_IMG}" "${MOUNT_POINT}"
sudo cp -a "${ROOTFS_DIR}/." "${MOUNT_POINT}/"
sudo umount "${MOUNT_POINT}"
rmdir "${MOUNT_POINT}"

# 步骤4：压缩空间（e2fsck + resize2fs）
echo "[4/4] 压缩镜像（可选）..."
e2fsck -p -f "${OUTPUT_IMG}" || true
resize2fs -M "${OUTPUT_IMG}"

echo ""
echo "[✓] 完成！输出文件: ${OUTPUT_IMG}"
echo "    大小: $(du -sh ${OUTPUT_IMG} | cut -f1)"
bash (宿主机)
chmod +x mk-ubuntu-rootfs.sh # 执行打包（需要 root 权限） sudo ./mk-ubuntu-rootfs.sh # 完成后查看输出 ls -lh ubuntu-rootfs.img
💡
镜像大小建议：
  • 命令行系统（无桌面）：IMG_SIZE_MB=2048 (2GB)
  • LXDE 轻量桌面：IMG_SIZE_MB=5120 (5GB)
  • GNOME 完整桌面：IMG_SIZE_MB=8192 (8GB)
注意：分区大小不能超过 eMMC 实际容量！ATK-DLRK3588 默认 eMMC 为 16GB。

集成到 SDK 并调整分区表

修改 parameter.txt 分区表

Ubuntu 根文件系统比 Buildroot 大很多，需要扩大 rootfs 分区。 原始分区表可能将 rootfs 限制在 4~8 GB，建议改为自动扩展（:grow）。

bash (宿主机)
cat ~/rk3588_sdk/device/rockchip/rk3588/parameter.txt

找到 mtdparts 行，将 rootfs 分区修改为使用剩余全部空间：

parameter.txt — 修改后示例
FIRMWARE_VER: 11.0 MACHINE_MODEL: RK3588 MACHINE_ID: 007 MANUFACTURER: RK3588 MAGIC: 0x5041524B ATAG: 0x00200800 MACHINE: 3588 CHECK_MASK: 0x80 PWR_HLD: 0,0,A,0,1 TYPE: GPT CMDLINE: mtdparts=:0x00002000@0x00004000(uboot),\ 0x00002000@0x00006000(misc),\ 0x00020000@0x00008000(boot),\ 0x00040000@0x00028000(recovery),\ -@0x00068000(rootfs:grow) uuid:rootfs=614e0000-0000-4b53-8000-1d28000054a9
⚠️
注意：修改 parameter.txt 时，不要修改 uuid 行（包括 uuid:rootfs= 后的值），否则系统将找不到 rootfs 分区！

替换 rootfs 镜像

bash (宿主机)
# 复制 Ubuntu rootfs 镜像到 SDK 的 images 目录 cp ~/ubuntu_build/ubuntu-rootfs.img \ ~/rk3588_sdk/buildroot/output/images/rootfs.img # 确认文件存在 ls -lh ~/rk3588_sdk/buildroot/output/images/rootfs.img

bash (宿主机)

重要：烧写顺序不可颠倒，必须先烧 Loader 再烧 parameter（分区表），然后才能烧写各分区镜像。 顺序错误或镜像不匹配可能导致开发板无法启动，需重新进入 Maskrom 模式重刷。

扩容根分区（重要！）

由于我们使用 :grow 参数，rootfs 分区会被分配到全部剩余空间，但文件系统本身还是打包时的大小，需要手动扩容：

bash (开发板上执行)
# 查看当前磁盘分区状态 lsblk df -h # 确认根分区设备名（通常是 /dev/mmcblk0p5 或 /dev/mmcblk1p5） ROOTDEV=$(findmnt -n -o SOURCE /) echo "根分区设备: ${ROOTDEV}" # 扩展文件系统到分区最大容量 sudo resize2fs ${ROOTDEV} # 验证扩容结果 df -h /

常见问题排查

现象可能原因解决方案
开机黑屏，无串口输出 Loader 或 U-Boot 烧录失败 重进 Maskrom 模式，重新烧录 MiniLoaderAll.bin 和 uboot.img
内核启动后无法挂载 rootfs rootfs.img 格式错误或分区表不匹配 检查 parameter.txt UUID 是否与 fstab 一致；重新打包 rootfs
系统启动后磁盘空间不足 未执行 resize2fs 执行 sudo resize2fs /dev/mmcblk0p5
网络无法连接 网卡驱动缺失或 NetworkManager 未管理 检查 ip link；重配 NetworkManager conf
图形界面黑屏 GPU 驱动未安装 / 显示管理器配置错 从 SDK buildroot target 复制 Mali libmali.so；检查 lightdm/gdm 日志
ping 报 "Operation not permitted" ping 权限问题 sudo setcap cap_net_raw+p /bin/ping
chroot 内 apt 无法连网 resolv.conf 未正确复制 sudo cp -L /etc/resolv.conf ubuntu_rootfs/etc/
内核模块加载失败 内核版本与模块版本不匹配 确保 modules_install 和 boot.img 使用同一内核源码编译

快速参考脚本汇总

将以下三个脚本放在同一目录，即可实现一键化操作流程：

① mount-rootfs.sh — 挂载/卸载工具

bash已在第4节完整给出
用法: sudo ./mount-rootfs.sh -m ubuntu_rootfs/ # 挂载 sudo ./mount-rootfs.sh -u ubuntu_rootfs/ # 卸载

② mk-ubuntu-rootfs.sh — 打包 ext4 镜像

bash已在第7节完整给出
用法: sudo ./mk-ubuntu-rootfs.sh # 输出: ubuntu-rootfs.img

③ flash-ubuntu.sh — 一键烧录脚本（Linux）

~/rk3588_sdk/flash-ubuntu.sh 
#!/bin/bash
# 一键烧录 Ubuntu 固件到 RK3588 开发板
set -e

SDK_DIR="$(cd "$(dirname "$0")" && pwd)"
TOOL="${SDK_DIR}/tools/upgrade_tool"
UBUNTU_ROOTFS="$HOME/ubuntu_build/ubuntu-rootfs.img"
IMG_DIR="${SDK_DIR}/buildroot/output/images"
PARAM="${SDK_DIR}/device/rockchip/rk3588/parameter.txt"
UBOOT_DIR="${SDK_DIR}/u-boot"

echo "=============================="
echo " RK3588 Ubuntu 一键烧录工具"
echo "=============================="

if [ ! -f "${UBUNTU_ROOTFS}" ]; then
    echo "[错误] 找不到 ${UBUNTU_ROOTFS}"
    echo "请先运行 mk-ubuntu-rootfs.sh 生成 rootfs 镜像"
    exit 1
fi

echo "[1/8] 检查设备连接..."
sudo ${TOOL} LD || { echo "未检测到设备，请确认开发板处于 Loader/Maskrom 模式"; exit 1; }

echo "[2/8] 烧写 Loader..."
sudo ${TOOL} UL ${UBOOT_DIR}/MiniLoaderAll.bin -noreset

echo "[3/8] 烧写分区表..."
sudo ${TOOL} DI -p ${PARAM}

echo "[4/8] 烧写 U-Boot..."
sudo ${TOOL} DI -uboot ${UBOOT_DIR}/uboot.img

echo "[5/8] 烧写 misc..."
sudo ${TOOL} DI -misc ${IMG_DIR}/misc.img

echo "[6/8] 烧写 boot（内核+设备树）..."
sudo ${TOOL} DI -boot ${IMG_DIR}/boot.img

echo "[7/8] 烧写 recovery..."
sudo ${TOOL} DI -recovery ${IMG_DIR}/recovery.img

echo "[8/8] 烧写 Ubuntu rootfs..."
sudo ${TOOL} DI -rootfs ${UBUNTU_ROOTFS}

echo ""
echo "[✓] 所有镜像烧写完成！正在重启开发板..."
sudo ${TOOL} RD
bash
chmod +x ~/rk3588_sdk/flash-ubuntu.sh
sudo ~/rk3588_sdk/flash-ubuntu.sh
